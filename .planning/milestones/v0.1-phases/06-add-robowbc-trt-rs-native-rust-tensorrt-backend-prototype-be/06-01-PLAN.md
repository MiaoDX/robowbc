# Phase 6 Plan 1

**Refined objective:** publish the provider-matched benchmark split in the
generated Markdown/HTML report and verify that the full site/report/browser
path still behaves correctly. This phase deliberately prioritizes the existing
HTML/report harness over a speculative `robowbc-trt-rs` prototype.

## Tasks

1. Render explicit path groups in the benchmark Markdown and HTML summary so
   planner-only, encoder+decoder-only, full velocity, and Decoupled rows are
   scannable without reading case IDs.
2. Update README and benchmark docs so provider truthfulness, blocked GPU rows,
   and the checked-in CPU default are explicit.
3. Rebuild the full showcase bundle and benchmark HTML with the existing site
   path, then validate the generated bundle.
4. Drive the generated GEAR-Sonic detail page in a real browser with the
   existing smoke harness to prove the final HTML/report surface shows no
   regression.

## Verification

- `python3 -m unittest tests.test_site_browser_smoke tests.test_policy_showcase tests.test_validate_site_bundle`
- `make showcase-verify PYTHON=python3 SITE_OUTPUT_DIR=/tmp/robowbc-site MUJOCO_DOWNLOAD_DIR=/tmp/mujoco`
- `make site-browser-smoke PYTHON=python3 SITE_OUTPUT_DIR=/tmp/robowbc-site SITE_BROWSER_POLICY=gear_sonic`

## Success Criteria

- The generated benchmark summary shows explicit path groups in Markdown and
  HTML.
- Public docs explain the new case split, provider contract, blocked-artifact
  behavior, and CPU default clearly enough to rerun locally.
- The full showcase/site bundle builds successfully with the benchmark HTML
  included.
- The browser smoke on the generated GEAR-Sonic page passes against the built
  HTML, confirming no regression in the final site surface.
